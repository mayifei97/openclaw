## 🤖 子 Agent 使用准则

**主动拆分任务原则(2026-04-27)**
- 批量处理/耗时独立/可并行/需不同模型 → 创建子agent，主agent负责调度汇总
- 完成后必须向主agent汇报；定期清理已完成子agent释放资源

**超时设置准则(2026-05-28)**
- 简单部署10-15min | 代码开发30-60min | 复杂开发分阶段每阶段30min
- 超时后创建新子agent继续，传递已有进度，不要主agent自己接手

**创建前检查准则(2026-05-07)**
- 创建前必须先查 `sessions_list`/`subagents list`，已有同功能子agent则不重复创建

**软件开发工程师子Agent(2026-04-27)**
- 角色：软件服务开发/维护/问题排查
- 服务范围：8887(Dashboard) | 8888(MrDoc v9.3) | 8889(文件管理器)
- 触发：用户提软件需求时分配给此子agent
- 原则：先分析→备份→修改→汇报，遇阻及时上报

**会话清理方法** → 详见 `memory/tech-ref.md`（3处清理：sessions.json索引 / session文件 / runs.sqlite数据库，勿自行restart网关）

### 操作纪律（2026-06-11回测验证版）

**核心原则：不同基金不同策略，强趋势只加不减，震荡型双向操作**

- 适用：人工智能(012733)、黄金ETF(000216)
- 🟢 偏离MA60 < -10% → 加仓500元（冷却5日）
- 🔴 偏离MA60 > +15% → 减仓1/3（冷却10日）
- ⚪ 其他 → 持有不动

**B类：震荡/均值回归基金（双向操作）**
- 适用：沪深300(460300)、富国天惠(161005)
- 🟢 偏离MA60 < -8% → 加仓500元（冷却5日）
- 🔴 偏离MA60 > +8% → 减仓1/3（冷却10日）
- ⚪ 其他 → 持有不动
