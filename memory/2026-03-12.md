## 🐛 Dashboard CPU 异常问题排查与修复 (09:00-09:30)

### 问题现象
- Dashboard 重启时经常导致 CPU 异常上升
- 需要重启系统才能解决
- 严重影响效率

### 根本原因分析
1. **代码重复**: `app.py` 中 `sysmon_data()` 函数有重复代码块
2. **多实例竞争**: `start_sysmon()` 检测不准确，可能启动多个 sysmon.js 实例
3. **文件竞争**: 多个 sysmon.js 同时读写 `sysmon_data.json`
4. **僵尸进程**: 旧进程未完全清理，新进程启动失败，反复尝试

### 修复内容

#### 1. app.py 修复
- 删除 `sysmon_data()` 中的重复代码块
- 重写 `start_sysmon()` 函数：
  - 启动前强制杀死所有旧进程
  - 清理损坏的数据文件
  - 使用 `start_new_session=True` 避免僵尸进程
  - 启动后验证进程状态

#### 2. sysmon.js 修复
- 添加单实例锁文件机制 (`sysmon.lock`)
- 启动时检查是否已有实例运行
- 退出时自动清理锁文件
- 防止多实例竞争数据文件

#### 3. 创建安全重启脚本
- 文件: `openclaw-dashboard/restart_dashboard.sh`
- 功能: 顺序停止进程 → 清理锁 → 等待端口 → 启动服务

### 后续操作规范

```bash
cd /home/admin/.openclaw/workspace/openclaw-dashboard
```

#### 实现效果
- ✅ 秒模式：每秒刷新，显示时:分:秒，实时滚动

- ✅ 切换模式：立即可见，无需等待，使用已有数据重新处理
