## 🐛 MrDoc API 上传问题 (01:00-01:15)

### 问题现象
- 使用 MrDoc API 上传 HTML 报告后，文档显示为源代码形式
- content 字段为 NULL，只有 md_content 有内容
- 页面无法正常渲染，只显示标题

### 根本原因
MrDoc v9.3 的 `create_doc` API 存在 bug：
- 无法正确设置 `content` 字段（始终为 NULL）
- 即使 `md_content` 有内容，页面也无法正确渲染
- 只有通过 Web 界面手动创建的文档才能正常显示

### 测试验证
| 文档ID | 格式 | content | md_content | 显示结果 |
|--------|------|---------|------------|----------|
| doc-53 | HTML | NULL | 11,879字节 | 正常显示 |
| doc-54 | HTML | NULL | 有内容 | 源代码 |
| doc-55 | HTML | NULL | 有内容 | 源代码 |
| doc-56 | HTML | NULL | 67,308字节 | 源代码 |
| doc-57 | HTML | NULL | 67,308字节 | 源代码 |
| doc-58 | HTML | NULL | 67,308字节 | 源代码 |
| doc-59 | Markdown | NULL | 有内容 | 源代码 |
| doc-60 | HTML | NULL | 0字节 | 空白 |
| doc-61 | HTML | NULL | 8,038字节 | 源代码 |
| doc-62 | Markdown | NULL | 0字节 | 空白 |

### 解决方案
**临时方案**：
1. 手动访问 MrDoc Web 界面
2. 新建文档 → 复制 HTML 内容 → 粘贴保存
3. 或使用 MrDoc 的导入功能

**长期方案**：
- 升级 MrDoc 到最新版本
- 或联系 MrDoc 开发者修复 API

### 用户反馈
"最后的 doc-61 这个生成的是对的了。你以后要记住生成觅思文档要按照这个格式，否则经常传上去的都是代码的形式。"

### 教训
- MrDoc API 有 bug，无法通过 API 正确上传 HTML 文档
- 以后生成报告后，需要手动上传到 MrDoc
- 已保存 HTML 文件到本地，方便手动上传
